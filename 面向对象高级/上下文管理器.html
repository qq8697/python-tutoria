# 上下文管理器

对于系统资源，如文件、数据库连接和 socket，程序打开这些资源并且执行完业务逻辑之后，必须做的一件事是关闭这些资源。

Python 中出现异常后如果未做处理会导致后序代码无法执行，以文件操作为例，可能会出现 close 方法无法被调用，导致对应文件描述符未关闭，一个进程中文件描述符数量有限，如果出现未关闭的情况，可能会导致后续程序没有文件描述符可用。

# 关闭文件的代码演进

  1. 常规方式：
def f1():
  f = open('t.txt', 'w')
  f.write('sth1')
  f.close()

# 上下文管理器

上下文管理器是一个对象，定义了使用 with 表达式时要建立的运行时上下文，处理执行代码块所需的运行时上下文的进入和退出（enter and exit）。白话文如下： 在执行语句体之前，先执行上下文管理器的“进入”方法，然后再执行语句体，最后执行“退出”方法。

  1. 保存和恢复全局状态；
  2. 锁定和解锁资源；
  3. 关闭打开的文件等。

# 实现 __enter__ 和 __exit__ 方法

任何实现了 __enter__ 和 __exit__ 方法的对象都可以称为上下文管理器。

示例，自定义 File 类：

class File(object):
  def __init__ (self, filename, mode):
    self.filename = filename
    self.mode = mode

  def __enter__(self):
    self.f = open(self.filename, self.mode)
    return self.f

  def __exit__(self, *args):
    self.f.close()

with File("t.txt", "w") as f:
  f.write("sth4")

# 使用 @contextmanager 装饰器

使用 @contextmanager 装饰器装饰函数，函数内 yield 之前在 __enter__ 中执行，yield 之后在 __exit__ 中执行。

示例，自定义 open 函数：

from contextlib import contextmanager

@contextmanager
def custom_open(path, mode):
  f = open(path, mode)
  yield f
  f.close()

with custom_open("t.txt", "w") as f:
  f.write("sth5")
